```javascript
export const CATEGORIES = ['context', 'amendment', 'comment'];

export class PromptManager {
    constructor();                                          // Initializes empty state, calls loadState()
    addPrompt(category, { name, template, description });   // Returns { id, name, template, description }
    deletePrompt(category, promptId);                       // Removes prompt, clears active if matched
    getPrompts(category);                                   // Returns prompt[] for category
    getPrompt(category, promptId);                          // Returns prompt or undefined
    selectPrompt(category, promptId);                       // Sets active (null to deactivate), returns prompt|null
    getActivePrompt(category);                              // Returns active prompt or null
    canSubmit();                                            // true if amendment OR comment is active
    getActiveMode();                                        // 'amendment' | 'comment' | 'both' | 'none'
    persistState(category);                                 // Writes to localStorage
    loadState();                                            // Reads from localStorage
    getState();                                             // Returns full state object
}
```

  <name>Task 2: Wire PromptManager into taskpane.js with tab switching, CRUD, activation, status updates, and dynamic button</name>
  <files>src/taskpane/taskpane.js</files>
  <action>
    Refactor `taskpane.js` to use the PromptManager from Plan 01 for all prompt operations. This is the largest task -- it replaces the existing prompt management code and adds tab switching, status updates, and dynamic button behavior.

    **1. Import PromptManager:**
    Add at top of file after existing imports:
    ```javascript
    import { PromptManager, CATEGORIES } from '../lib/prompt-manager.js';
    ```

    **2. Replace global prompt state:**
    Remove `let prompts = [];` (line 16). Add:
    ```javascript
    const promptManager = new PromptManager();
    let currentTab = 'context';
    const unsavedText = { context: '', amendment: '', comment: '' };
    ```
    The `unsavedText` object preserves textarea content across tab switches (per Pitfall 2 from research).

    **3. Remove old prompt functions:**
    Delete: `initializeDefaultPrompts()`, `loadPrompts()`, `renderPrompts()`, `handlePromptSelect()`, `showSavePromptModal()`, `hideSavePromptModal()`, `handleSavePrompt()`, `handleDeletePrompt()`, `handleResetPrompt()`.

    **4. Add new prompt functions:**

    `renderCategoryDropdown(category)` -- Populates `#promptSelect-{category}` with `(None)` option + prompts from `promptManager.getPrompts(category)`. Selects the active prompt if one exists.

    `renderAllDropdowns()` -- Calls renderCategoryDropdown for each category.

    `handleCategoryPromptSelect(category, promptId)` -- If promptId is empty string (the "(None)" option), call `promptManager.selectPrompt(category, null)`, clear textarea, set `unsavedText[category] = ''`. Otherwise call `promptManager.selectPrompt(category, promptId)`, load template into textarea, set `unsavedText[category] = prompt.template`. Call `updateDotIndicators()`, `updateStatusSummary()`, `updateReviewButton()`.

    `switchTab(category)` -- Save current textarea value to `unsavedText[currentTab]`. Update `currentTab = category`. For each category: set tab button `aria-selected`, `classList.active`, `tabIndex`, panel `hidden`. Restore textarea from `unsavedText[category]`.

    `handleTabKeydown(e)` -- Arrow key navigation within tablist per WAI-ARIA pattern. ArrowRight/ArrowLeft cycle through tabs, Home/End jump to first/last. Prevent default, focus new tab, call switchTab.

    `updateDotIndicators()` -- For each category, toggle `.active` class on `#dot-{category}` based on `promptManager.getActivePrompt(category) !== null`.

    `updateStatusSummary()` -- For each category, update the `.status-dot` class and `.status-value` text in `#promptStatusSummary`. Active: green dot + prompt name. Inactive: open circle + "(none)".

    `updateReviewButton()` -- Per user's locked decision. Check `promptManager.getActiveMode()`:
    - 'amendment': label "Amend Selection", enabled, clear title
    - 'comment': label "Comment on Selection", enabled, clear title
    - 'both': label "Amend & Comment", enabled, clear title
    - 'none': label "Review Selection", disabled, title "Select an Amendment or Comment prompt to enable"
    Use arrow character (Unicode 2192) at end of label.

    `showSavePromptModal(category)` -- Open modal, set `#savePromptCategory` text to "Saving to: {Category}" (capitalize first letter), clear name/description inputs, store `category` for confirm handler.

    `handleSavePromptConfirm()` -- Read name, description, template from `#promptTextarea-{currentTab}`. Call `promptManager.addPrompt(currentTab, {name, template, description})`. Render dropdown, hide modal, select the new prompt, log success.

    `handleDeletePromptConfirm(category)` -- Get selected promptId from dropdown. Call `promptManager.deletePrompt(category, promptId)`. Render dropdown, clear textarea, update dots/status/button. Log success.

    `handleResetPrompt(category)` -- Clear textarea for category, clear `unsavedText[category]`. Do NOT deactivate (just clears the editing area). User can select (None) to deactivate.

    **5. Update initialize():**
    Replace the old prompt setup event listeners. Wire:
    - Tab buttons: click -> switchTab(category), keydown -> handleTabKeydown
    - Per-category dropdowns: change -> handleCategoryPromptSelect(category, e.target.value)
    - Per-category save buttons: click -> showSavePromptModal(currentTab)
    - Per-category delete buttons: click -> handleDeletePromptConfirm(currentTab)
    - Per-category clear buttons: click -> handleResetPrompt(currentTab)
    - Status summary lines: click -> switchTab(data-category)
    - Modal confirm: click -> handleSavePromptConfirm()
    - Modal cancel: click -> hideSavePromptModal()

    Remove the old `loadPrompts()` call. The PromptManager constructor already calls `loadState()`.

    Call `renderAllDropdowns()`, `updateDotIndicators()`, `updateStatusSummary()`, `updateReviewButton()` at end of initialize().

    **6. Update handleReviewSelection():**
    Change prompt text retrieval from `document.getElementById("promptTextarea").value` to reading from the PromptManager's active amendment prompt template (fall through to Plan 03 for full composition). For NOW (Plan 02), keep the existing single-prompt-to-LLM flow but read from `promptManager.getActivePrompt('amendment')?.template` instead of the textarea. This maintains the existing amendment workflow while Plan 03 adds full composition.

    **7. Keep textarea as editing view:**
    The textarea is a VIEW of the selected prompt's template. When the user edits it, that edit is tracked in `unsavedText[category]` and survives tab switches. The user must click Save to persist edits to the library. The saved library (via PromptManager) is the source of truth for what gets sent to the LLM.

    **Important conventions:**
    - 4-space indentation, semicolons, camelCase
    - Keep addLog calls for user feedback on prompt operations
    - Keep the isProcessing guard in handleReviewSelection
    - Do NOT touch settings, connection, verification, or logging code sections
