  <behavior>
    - State init: PromptManager initializes with three categories (context, amendment, comment), each with empty prompts array and null activePromptId
    - CRUD - addPrompt(category, {name, template, description}): generates ID from name, adds to category's prompts array, returns the prompt object
    - CRUD - addPrompt with duplicate ID: updates existing prompt (upsert behavior)
    - CRUD - deletePrompt(category, promptId): removes prompt from array; if deleted prompt was active, sets activePromptId to null
    - CRUD - getPrompts(category): returns the prompts array for that category
    - CRUD - getPrompt(category, promptId): returns single prompt or undefined
    - Activation - selectPrompt(category, promptId): sets activePromptId for that category, returns the prompt
    - Activation - selectPrompt(category, null): deactivates category (sets activePromptId to null)
    - Activation - selectPrompt auto-activates: selecting a prompt ID that exists sets it active
    - Activation - getActivePrompt(category): returns the active prompt object or null
    - Validation - canSubmit(): returns true if amendment OR comment has an active prompt, false if neither
    - Validation - getActiveMode(): returns 'amendment' | 'comment' | 'both' | 'none' based on active state
    - Context optional: canSubmit() does NOT require context to be active
    - Persistence - persistState(category): writes category's prompts array to `wordAI.prompts.{category}` and activePromptId to `wordAI.active.{category}`
    - Persistence - loadState(): reads all three categories from localStorage, falls back to empty arrays if missing
    - Persistence - loadState with corrupted JSON: falls back to empty state without throwing
    - No migration: loadState() does NOT read from old `wordAI.prompts` key; starts fresh
    - ID generation: name.toLowerCase().replace(/\s+/g, '-') per existing codebase pattern
  </behavior>
